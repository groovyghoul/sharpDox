Erstellen von erweiterten Dokumentationen

Version 0.9.8

Artikel

Neben der eigentlichen Code-Dokumentation können Artikel, erweiterte Beschreibungen zu Namensräumen und benutzerdefinierte Navigationen erstellt werden.

Erweiterte Dokumentationen können aus den folgenden Elementen bestehen:

Code-Dokumentation

Die Kern Aufgabe von sharpDox besteht darin ein Code-Dokumentation aus den Kommentaren einer C#-Solution zu erstellen. Wird innerhalb des Solutionpfads keine benutzerdefinierte Navigation gefunden, erstellt sharpDox eine einfach Code-Dokumentation.

Innerhalb einer komplexen Dokumentation kann die Code-Dokumentation in der benutzerdefinierten Navigation an einer beliebigen Stelle angehängt werden (siehe Punkt: Navigation).

Projektbeschreibung

Es kann eine Projektbeschreibung angelegt werden, die vom CHM- & HTML-Exporter als Text für die Startseite genutzt wird. Diese Beschreibung kann in der Markdown-Syntax geschrieben werden.

Dazu muss eine Datei mit dem Namen README.md im Solutionpfad liegen. Diese Datei wird während des Bauvorgangs eingelesen und den Exportern übergeben.

Erstellen einer erweiterten Projektbeschreibung

Lege eine Datei README.md im Solutionpfad an und nutze die Markdown-Syntax für Textformatierungen.

Namensraum-Beschreibungen

sharpDox ermöglicht es auch für Namensräumen komplexere Beschreibungen zu erstellen. Dazu innerhalb des Solutionordners - Unterordner funktionieren ebenfalls - eine Datei mit folgendem Format erstellt werden: Namens.raum.Name.sdn

Erstellen von Namensraum-Beschreibungen

Lege eine Datei mit der Endung .sdnd und dem Namen des Namensraum innerhalb des Solutionordners an. Für die Textformatierung kann die Markdown-Syntax genutzt werden. Bespiel: SharpDox.Sdk.sdnd - Eine Beschreibung für den Namensraum SharpDox.Sdk.

Artikel

Neben Beschreibungen und der Dokumentation des Codes können in sharpDox auch Artikel Bestandteil einer Dokumentation sein.

Artikel sind dabei zusätzliche Seiten innerhalb der Dokumentation, die über eine benutzerdefinierte Navigation erreichbar sind. Für Artikel ist es zwingend notwendig, dass eine Navigation erstellt wurde. Ansonsten werden Artikel während des Bauvorgangs nicht berücksichtigt.

Um einen Artikel, in einer Markdown-Syntax, zu erstellen, muss lediglich eine Datei mit der Endung .sda im Solutionordner erstellt werden. sharpDox erkennt diese Dateien und liest sie während des Bauvorgangs ein. Siehe Navigation für Informationen wie diese Artikel verlinkt werden können.

Erstellen von Artikeln

Lege eine Datei mit der Endung .sda innerhalb des Solutionordners an. Der Name der Datei ist beliebig. Für die Textformatierung kann die Markdown-Syntax genutzt werden.

Links

Innerhalb von Beschreibungen und Artikeln können auf verschiedene Elemente innerhalb der Dokumentation verlinkt werden. Außerdem ist es möglich auf Bilder innerhalb der Lösung zu verlinken.

Die Platzhalter werden statt einer URL genutzt (In den Beispielen wird die Markdown-Syntax genutzt):

Namensraum - namespace/[Fullname].html

Beispiel: [LinkText](namespace/0c303f67-cd1f-4d26-84ff-0cc23fe2a32a.html)

Typ - type/[Identifier].html

Beispiel: [LinkText](type/4dd5707c-4a10-490b-9267-147133d41d74.html)

Artikel - article/[ArticleName].html

Beispiel: [LinkText](article/exporter-tutorial.html)

Bilder - Innerhalb der Solution können Bilder abgelegt werden die in Beschreibungen und Artikeln genutzt werden können. Die Bilder für die Dokumentation müssen mit dem Prefix sdi.* beginnen.

Beispiel: ![Shell](images/sdi.startSD.png)

Navigation

Das wichtigste Element einer erweiterten Dokumentation ist die Navigation. Mit dieser kann festgelegt werden wie die Struktur der Dokumentation aufgebaut ist. Als Beispiel soll die Navigation der sharpDox Dokumentation genommen werden:

-Einführung#introduction
-Benutzer
--Erstellen von erweiterten Dokumentationen#advanced-tutorial
--Erstellen von mehrsprachigen Dokumentationen#multilang-tutorial
-Entwickler
--Erstellen eines Exporters#exporter-tutorial
--API#SDDoc

Die Anzahl der anfänglichen Bindestriche gibt die Dokumentationsebene an. Gefolgt vom Namen des Menüpunktes. -Einführung bedeutet also, dass der Menüpunkt mit dem Namen Einführung an erster Stelle auf der obersten Ebene erstellt werden soll. Getrennt durch ein # folgt darauf der Name des Artikels, ohne Dateiendung, auf den der Menüpunkt verlinken soll. Ist kein Link angegeben, dient der Menüpunkt lediglich als "Platzhalter" (zum Beispiel die zweite Zeile -Benutzer).

--Erstellen von erweiterten Dokumentationen#advanced-tutorial

Bedeutet also, dass sharpDox auf der zweiten Ebene, unterhalb des darüberliegenden Elements, einen Menüpunkt mit dem Namen Erstellen von erweiterten Dokumentationen erstellt, der auf den Artikel advanced-tutorial verlinkt.

Soll ein Menüpunkt auf die von sharpDox erstellte Code-Dokumentation verlinken wird stat eines Artikelnamens lediglich das Schlüsselwort SDDoc genutzt (Im Beispiel: --API#SDDoc).

Die Navigationsdatei muss ebenfalls im Solutionpfad liegen und den Namen default.sdnav tragen.
