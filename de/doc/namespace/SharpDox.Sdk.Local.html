SharpDox.Sdk.Local

Assembly: SharpDox.Sdk

Beschreibung

Der Local-Namensraum beinhaltet die Interfaces, um eigene Strings für sharpDox zu erstellen. Die Standardsprache in sharpDox ist Englisch.

Beim Start von sharpDox wird für jede registrierte Sprachdatei eine Datei im lang/defaults Ordner angelegt. Zum Beispiel die Datei en.SharpDox.sdlang, die alle Strings beinhaltet die im Core von sharpDox verwendet werden. Falls diese Strings zum Beispiel in das Spanische übersetzt werden sollen, genügt es eine Kopie der Datei in den Ordner lang abzulegen. Diese Datei muss daraufhin in es.SharpDox.sdlang umbenannt werden. Jede neue Übersetzung muss mit dem zweistelligen ISO-Code der jeweiligen Sprache beginnen. Die Strings in dieser Datei können jetzt in das Spanische übersetzt werden.

Falls die Systemsprache des Hosts auf Spanisch gesetzt ist, wird sharpDox für die Übersetzung alle es.* Dateien nehmen, um eine entsprechende Übersetzung anzuzeigen. Falls keine Übersetzung vorhanden ist, werden die ursprünglichen Strings genutzt.

Hier ein Beispiel, wie eine Sprachdatei mit einem String implementiert werden kann:

using SharpDox.Sdk.Local;

namespace ExampleExporter
{
    public class ExampleStrings : ILocalStrings
    {
        private string _exampleString = "English String";

        public string ExampleString
        {
            get { return _exampleString; }
            set { _exampleString = value; }
        }

        public string DisplayName { get { return "ExampleExporter"; } }
    }
}

Die Benutzung dieser Strings gestaltet sich analog zu der Benutzung von konfigurationssektionen. Siehe SharpDox.Sdk.Config die beschreibung.

Typen

Name Description
ILocalStrings

Jedes Plugin, das lokalisierte Strings nutzen möchte, muss dieses Interface implementieren.
SharpDoxStrings
